# TA选课系统使用手册

适用角色：

- 学生（TA）
- TA Admin
- Course Admin

本手册基于当前系统最新实现整理，覆盖：

- 本地登录与 SSO 登录
- 学生申请主流程
- TA Admin 初审与发布至教授
- Course Admin 教学班、人员、申请、报表、审计

## 1. 系统入口

系统支持两种登录方式：

- 本地账号密码登录
- SSO 统一身份认证登录

常见入口示例：

- 本机访问：`http://127.0.0.1:3000`
- 局域网访问：`http://<内网IP>:3000`
- 服务器访问：`http://<公网IP>:<端口>` 或正式 HTTPS 地址

## 2. 登录说明

登录页包含：

- SAIF 品牌页头
- 系统简介区
- 登录表单
- `登录`
- `SSO 登录`

登录成功后，系统会根据账号角色自动进入对应首页。

## 3. 角色总览

### 3.1 学生（TA）

学生可以：

- 查看可申请教学班
- 查看冲突信息
- 提交 TA 申请
- 查看自己的申请状态
- 撤销尚未进入 Professor 审批阶段的申请
- 维护个人简历

### 3.2 TA Admin

TA Admin 可以：

- 查看待初审申请
- 单条或批量审批
- 查看全部申请
- 查看全部教学班
- 审核某个教学班下的申请
- 生成并发送 Professor 邮件
- 管理 TA 申请资格

### 3.3 Course Admin

Course Admin 可以：

- 管理教学班
- 创建、编辑、删除教学班
- 导入教学班与排课
- 管理人员
- 导入人员
- 查看全部申请与申请日志
- 查看报表与审计日志

## 4. 学生（TA）使用说明

### 4.1 可申请教学班

学生登录后进入 `可申请教学班` 页面。

页面支持：

- 按 `是否可申请`
- 按 `教授名`
- 按 `课程名称`
- 按 `教学班名称`
- 按 `授课语言`

教学班卡片当前会展示：

- 教学班名称
- 教授
- 授课语言
- 学分
- 当前状态
- `待初审申请数`

说明：

- 页面不再展示“已通过人数”
- 页面不再展示“学期”
- `待初审申请数` 表示当前该教学班中，状态为 `待 TAAdmin 审批` 的申请数量
- 如果该教学班的上一条申请已被拒绝，学生会看到 `被拒绝` 状态，并可再次申请
- 被拒绝的历史申请不会再参与新的冲突阻断

示意图：

![学生可申请教学班页面](screenshots/ta_classes_current.png)

### 4.2 教学班详情与提交申请

点击 `查看详情` 后，学生可以查看：

- 教学班基础信息
- 排课安排
- 冲突信息
- 当前简历
- 提交申请区域

当前规则：

- `申请原因` 为非必填
- `简历` 为必填
- 如教学班不允许冲突申请，且存在阻断性冲突，则无法提交
- 提交后按钮会暂时变灰，避免重复点击

### 4.3 我的申请

在 `我的申请` 页面，学生可以查看自己的全部申请。

页面包含：

- 申请列表
- 教授列
- 学分列
- 申请时间
- 当前状态
- TAAdmin 备注
- Professor 备注
- 操作按钮

顶部摘要卡会展示：

- 待审批教学班数
- 待审批申请学分
- 已通过教学班数
- 已通过教学学分合计

按钮说明：

- `详情`
- `撤销`

说明：

- `撤销` 会先弹出确认框
- 已撤销、被拒绝的申请可以再次申请
- 如果申请已经进入更后阶段，是否允许撤销会按系统状态控制

示意图：

![学生我的申请页面](screenshots/ta_my_applications.png)

### 4.4 个人资料

在 `个人资料` 页面，学生可以：

- 查看姓名、账号、邮箱
- 上传个人简历
- 更新个人简历

说明：

- 简历支持中文文件名
- 更新后，后续申请与已有关联申请会显示最新简历

## 5. TA Admin 使用说明

### 5.1 待初审申请

在 `待初审申请` 页面，TA Admin 可以按条件筛选待审核申请，并执行单条或批量审批。

筛选维度：

- 教学班
- 教授
- 申请学生

可执行动作：

- 单独审批
- 批量通过
- 批量拒绝

说明：

- `审批备注` 现在为非必填
- TA Admin 通过后，申请状态进入 `待教授审批`
- TA Admin 拒绝后，状态进入 `TAAdmin 拒绝`

示意图：

![TAAdmin待初审申请页面](screenshots/taadmin_pending_current.png)

### 5.2 按教学班审核申请

在 `全部教学班` 中点击 `审核`，可进入某个教学班下的申请审核页。

页面可查看：

- 该教学班全部申请
- 冲突教学班摘要
- 简历
- TAAdmin
- 当前状态

### 5.3 全部申请

在 `全部申请` 页面，TA Admin 可以查看所有 TA 申请。

筛选维度：

- 教学班
- 教授
- 申请学生
- 状态

用途：

- 追踪全量申请状态
- 回看历史审批
- 进入详情页查看日志和备注

### 5.4 全部教学班

在 `全部教学班` 页面，TA Admin 可以按教学班维度查看初审工作。

页面包含：

- 教学班筛选
- 日历视图
- 导出教学班 TA
- 审核入口
- 批量设置开放时间
- 批量更新开放设置
- 批量修改发布状态

当前主要字段包括：

- 教学班代码
- 教学班名称
- 学分
- 教授
- 学期
- 开放状态
- TA已满
- 排课数
- TA
- 已通过/上限
- 申请数
- 待 TAAdmin 审批数
- 发布至教授

说明：

- `课程名` 已隐藏
- 点击 `排课数` 即可查看排课安排
- `发布至教授 = 已发送` 后，教学班会自动关闭申请
- 此时 `开放申请 = N`，学生不能再申请

示意图：

![TAAdmin全部教学班页面](screenshots/taadmin_classes_current.png)

### 5.5 发布至教授与邮件预览

TA Admin 在 `全部教学班` 中勾选教学班后，可以：

- 生成邮件预览
- 发送邮件给 Professor
- 抄送当前 TA Admin

系统行为：

- 按教授分别生成邮件
- 若一个教学班有多个教授，则每位教授都能收到
- 发送成功后，`发布至教授` 更新为 `已发送`
- 教授端会同时收到站内通知

示意图：

![TAAdmin邮件预览页面](screenshots/taadmin_email_preview.png)

### 5.6 TA 管理

在 `TA 管理` 页面，TA Admin 可以：

- 查看 TA 列表
- 查看申请数
- 查看已通过数
- 切换是否允许申请

示意图：

![TAAdmin TA管理页面](screenshots/taadmin_manage_current.png)

## 6. Course Admin 使用说明

### 6.1 教学班管理

`教学班管理` 是 Course Admin 最核心的页面之一。

页面支持：

- 教学班筛选
- 展开批量操作
- 展开新增教学班
- 导入教学班与排课
- 日历视图
- 导出教学班 TA

当前新增/编辑教学班支持维护：

- 教学班代码
- 教学班缩写
- 教学班名称
- 学分（浮点数）
- 授课语言
- Professor（可多选）
- 学期
- 最大录取人数
- 是否开放申请
- 是否允许冲突
- 开放开始时间
- 开放结束时间
- 课程介绍
- 备注
- 排课记录

说明：

- 导入功能也支持 `学分`
- 列表中隐藏了 `课程名`
- `排课数` 可直接点击查看排课
- 操作按钮保持单行展示

示意图：

![CourseAdmin教学班管理页面](screenshots/courseadmin_classes_current.png)

### 6.2 人员管理

在 `人员管理` 页面，Course Admin 可以：

- 查看所有人员
- 按姓名、登录名、邮箱、角色、是否允许申请筛选
- 排序
- 新增人员
- 编辑人员
- 删除人员
- 导入人员

角色常见包括：

- `TA`
- `Professor`
- `TAAdmin`
- `CourseAdmin`

示意图：

![CourseAdmin人员管理页面](screenshots/courseadmin_users_current.png)

### 6.3 全部申请

在 `全部申请` 页面，Course Admin 可以：

- 按教学班、教授、学生、状态查看申请
- 打开单条申请详情
- 查看审批记录
- 查看申请业务日志
- 手工修改申请状态

管理员改状态后：

- 会保留审批日志
- 会新增业务日志/审计日志

示意图：

![CourseAdmin全部申请页面](screenshots/courseadmin_applications_current.png)

### 6.4 申请日志

在 `申请日志` 页面，Course Admin 可以按更管理化的视角排查申请轨迹。

筛选维度：

- 申请学生
- 教学班
- 教授
- 状态
- 申请时间起止

每一行会展示：

- 当前状态
- 最新动作
- 最近几条业务日志摘要

### 6.5 报表视图

在 `报表视图` 页面，Course Admin 可以查看：

- 申请总数
- 待 TAAdmin 审批
- 待 Professor 审批
- 已通过
- 已拒绝
- 已撤销
- 开放中教学班
- TA 已满教学班
- 申请 TA 人数
- 通过率

页面还提供：

- 热门教学班
- 教授维度申请分布
- 申请状态概览
- 名额使用率 Top N 教学班
- 最近 7 日申请趋势
- 导出 Excel

示意图：

![CourseAdmin报表视图页面](screenshots/courseadmin_reports_current.png)

### 6.6 审计日志

在 `审计日志` 页面，Course Admin 可以查看系统级操作痕迹。

可筛选：

- 操作人
- 动作类型
- 对象类型
- 关键字

系统当前已经覆盖的审计范围包括：

- 学生提交申请
- 学生撤销申请
- TAAdmin 审批
- Professor 审批
- 管理员改状态
- 教学班创建/编辑/删除
- 人员创建/编辑/删除
- 教学班导入
- 人员导入
- 邮件发送及失败场景

## 7. 状态说明

常见申请状态：

- `待 TAAdmin 审批`
- `TAAdmin 拒绝`
- `待教授审批`
- `Professor 拒绝`
- `已通过`
- `已撤销`

补充说明：

- 被拒绝的教学班可以重新申请
- 被拒绝的历史申请不会继续参与阻断性冲突判断

## 8. 通知与邮件

当前系统同时支持：

- 站内通知
- 邮件通知

通知触发示例：

- 学生提交申请 -> 通知 TAAdmin
- TAAdmin 审批 -> 通知学生
- Professor 审批 -> 通知学生
- 发布至教授 -> 邮件 + 站内通知给 Professor

## 9. 使用建议

推荐的日常使用顺序：

### 学生（TA）

1. 先上传个人简历
2. 浏览可申请教学班
3. 查看冲突并提交申请
4. 在我的申请中跟踪状态

### TA Admin

1. 在待初审申请中完成初审
2. 在全部教学班中按班级维度复核
3. 生成并发送 Professor 邮件
4. 跟踪后续申请状态

### Course Admin

1. 维护教学班和人员
2. 管理开放时间和开放规则
3. 通过全部申请和申请日志跟踪流程
4. 用报表视图与审计日志做管理复盘
